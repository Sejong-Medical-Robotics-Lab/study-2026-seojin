### 실험 폴더 자동 생성 스크립트 (new_exp.sh) 사용법
- 스크립트 파일 저장 위치: ~/robot_study/week1/new_exp.sh
- alias 등록해둠 → 어느 폴더에 있든 `newexp`만 치면 실행됨
  (등록 명령: echo "alias newexp='~/robot_study/week1/new_exp.sh'" >> ~/.bashrc)
**사용법**: 
cd ~/robot_study/(원하는 주차폴더)
newexp 실험이름

### 미션 ② 수정–확인–커밋 사이클 에서
①: 수정 전 깨끗한 상태("nothing to commit, working tree clean")
	- 로컬과 GitHub가 완전히 동기화 됨. 아무런 변경사항 없음
② 파일 수정 후, add 전("Changes not staged for commit:" + modified(빨간색) )
	-파일은 수정됐지만 아직 스테이지에 안 올라간 상태
③ add 후, commit 전("Changes to be committed" + modified/new file(초록색)_
	-스테이지에 올라옴 다음 커밋에 포함될 준비 완료한 상태
